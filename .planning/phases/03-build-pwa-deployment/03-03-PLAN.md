  ## MovingCap Static Site
  Brief: Product catalog and motion calculator for MovingCap servo drives. React SPA with PWA offline support.

  ## Features
  - Product catalog (7 servo drive products with specs and datasheets)
  - S-curve motion profile calculator with interactive charts
  - Multi-language support (EN, DE, IT, FR)
  - PWA — installable, works offline after first visit
  - Responsive design (desktop + mobile)

  ## Prerequisites
  Step-by-step for IT admin audience:
  - Node.js 18 or newer (link to https://nodejs.org/, recommend LTS)
  - npm (comes with Node.js)
  - How to verify: `node --version` and `npm --version`

  ## Quick Start (Development)
  ```bash
  npm install
  npm run dev
  ```
  Opens at http://localhost:5173

  ## Build for Production
  ```bash
  npm run build
  ```
  Output in `dist/` folder — copy entire folder to web server.

  ### Custom Base Path (Subfolder Deployment)
  If deploying to a subfolder like `https://example.com/movingcap/`:
  ```bash
  # Linux/Mac:
  VITE_BASE_URL=/movingcap/ npm run build

  # Windows PowerShell:
  $env:VITE_BASE_URL="/movingcap/"; npm run build

  # Windows CMD:
  set VITE_BASE_URL=/movingcap/ && npm run build
  ```
  Or create a `.env` file in the project root:
  ```
  VITE_BASE_URL=/movingcap/
  ```

  ### Preview Build Locally
  ```bash
  npm run preview
  ```

  ## Deploy to Apache
  Step-by-step:
  1. Build the project (see above)
  2. Copy the entire `dist/` folder contents to your Apache document root or subfolder
  3. No `.htaccess` rewrite rules needed — the app uses hash-based routing (`#/products`, `#/calculator`)
  4. Optional: Add cache headers for static assets

  Example Apache directory structure:
  ```
  /var/www/html/movingcap/
  ├── index.html
  ├── assets/
  ├── images/
  ├── icons/
  ├── sw.js
  └── manifest.webmanifest
  ```

  ### HTTPS Requirement for PWA
  PWA features (offline mode, install prompt) require HTTPS. If served over HTTP or file://, the app still works but without offline caching.

  ## PWA / Offline
  - After first visit over HTTPS, the app caches all assets for offline use
  - Updates are applied silently on next visit when a new version is deployed
  - A subtle "Offline" indicator appears when network is unavailable
  - External PDF datasheets (hosted on movingcap.de) are not available offline

  ## Project Structure
  Brief overview of key directories:
  ```
  src/
  ├── pages/          # Page components (Landing, Products, Calculator, etc.)
  ├── components/     # Shared components, UI library, data
  ├── Layout.jsx      # App shell (header, footer, navigation)
  └── App.jsx         # Router configuration
  public/
  ├── images/         # Product images
  └── icons/          # PWA icons and favicons
  ```

  ## Tech Stack
  - React 18 + React Router 6 (HashRouter)
  - Vite 6 (build tool)
  - Tailwind CSS 3.4 + shadcn/ui components
  - Framer Motion (animations)
  - Recharts (calculator charts)
  - vite-plugin-pwa (service worker + manifest)

  ## Troubleshooting
  Common issues:
  - "Assets not loading after deploy" → Check VITE_BASE_URL matches your deployment path
  - "PWA not working" → Ensure HTTPS; file:// and HTTP don't support service workers
  - "Old version still showing" → Service worker updates on next visit; hard-refresh (Ctrl+Shift+R) to force

  Keep the tone practical and direct. No marketing fluff. This is an internal deployment guide for the IT team.

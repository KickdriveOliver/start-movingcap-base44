<task type="auto">
  <name>Task 2: Fix sonner.jsx — remove next-themes dependency</name>
  <files>src/components/ui/sonner.jsx</files>
  <action>
Replace the `next-themes` import and `useTheme()` call with a hardcoded theme value. The sonner component is a shadcn/ui component that's currently unused (App.jsx uses the Radix toast), but we keep it functional for potential future use.

Replace the full file content with:

```jsx
"use client";
import { Toaster as Sonner } from "sonner"

const Toaster = ({
  ...props
}) => {
  return (
    (<Sonner
      theme="light"
      className="toaster group"
      toastOptions={{
        classNames: {
          toast:
            "group toast group-[.toaster]:bg-background group-[.toaster]:text-foreground group-[.toaster]:border-border group-[.toaster]:shadow-lg",
          description: "group-[.toast]:text-muted-foreground",
          actionButton:
            "group-[.toast]:bg-primary group-[.toast]:text-primary-foreground",
          cancelButton:
            "group-[.toast]:bg-muted group-[.toast]:text-muted-foreground",
        },
      }}
      {...props} />)
  );
}

export { Toaster }
```

Changes: Removed `import { useTheme } from "next-themes"`, removed `const { theme = "system" } = useTheme()`, hardcoded `theme="light"`. The app has no dark mode — light is correct.

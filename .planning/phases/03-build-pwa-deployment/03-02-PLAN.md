<task type="auto">
  <name>Task 1: Install vite-plugin-pwa and configure PWA</name>
  <files>vite.config.js, package.json</files>
  <action>
  1. Run `npm install --save-dev vite-plugin-pwa`

  2. Update `vite.config.js` to add the VitePWA plugin with full configuration:
     - Import: `import { VitePWA } from 'vite-plugin-pwa'`
     - `registerType: 'autoUpdate'` — silent auto-update
     - `workbox.globPatterns`: `['**/*.{js,css,html,png,jpg,jpeg,svg,ico,woff,woff2}']`
     - `includeAssets`: `['favicon.svg', 'favicon.ico', 'icons/safari-pinned-tab.svg']`
     - `manifest` object with:
       - `name`: 'MovingCap - Servo Drive Products'
       - `short_name`: 'MovingCap'
       - `description`: 'Product catalog and motion calculator for MovingCap servo drives'
       - `theme_color`: '#1e40af' (blue-800, matches primary brand)
       - `background_color`: '#ffffff'
       - `display`: 'standalone'
       - `start_url`: './'
       - `scope`: './'
       - `icons`: array with android-chrome-192x192.png (192x192), android-chrome-512x512.png (512x512), and a maskable entry for 512x512
     - Keep existing `base`, `plugins: [react()]`, and `resolve.alias` config

  3. Do NOT remove the existing `<link rel="icon">` from index.html — that's the favicon, not the manifest.
  </action>
  <verify>Run `npm run build`. Verify `dist/manifest.webmanifest` exists and contains correct app name, icons, and theme_color. Verify `dist/sw.js` exists. Verify `dist/index.html` contains a `<link rel="manifest">` tag injected by the plugin.</verify>
  <done>vite-plugin-pwa installed, manifest generated with full branding, service worker generated with precache</done>
</task>

<task type="auto">
  <name>Task 2: Create OfflineIndicator component</name>
  <files>src/components/OfflineIndicator.jsx</files>
  <action>
  Create a new component `src/components/OfflineIndicator.jsx` that:

  1. Uses `useState` to track `isOffline` state (initialized from `!navigator.onLine`)
  2. Uses `useEffect` to listen for `online`/`offline` events on `window`
  3. Returns `null` when online
  4. When offline, renders a subtle fixed-position indicator:
     - Small pill/badge at the bottom-center of the viewport
     - Uses existing shadcn/Tailwind classes for consistency
     - Text: "Offline" with a small wifi-off icon from lucide-react
     - Semi-transparent background, subtle shadow
     - `z-50` to appear above content but not block interaction
     - Animate in/out with a simple transition (use Tailwind transition classes, not framer-motion to keep it lightweight)

  Keep it simple — this is a status indicator, not a banner.
  </action>
  <verify>Component file exists and exports a default React component. No import errors.</verify>
  <done>OfflineIndicator component created with navigator.onLine detection and subtle UI</done>
</task>

<task type="auto">
